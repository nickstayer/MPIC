# MPIC - Монитор Интеграции Почты и CRM

## Назначение

Приложение `MPIC` (Mail-CRM Integration Checker) представляет собой службу мониторинга, предназначенную для проверки корректности работы интеграции между почтовым ящиком и CRM-системой "Мегаплан".

Основная задача — автоматически отслеживать, создаются ли в Мегаплане сделки после получения писем от клиентов, и оперативно уведомлять ответственных лиц о сбоях и восстановлении работы интеграции.

## Принцип работы

1.  **Периодическая проверка**: Приложение запускается через заданные интервалы времени.
2.  **Анализ почты**: На каждом цикле оно подключается к указанному в настройках почтовому ящику (по протоколу IMAP) и получает информацию о последнем входящем письме.
3.  **Запрос к CRM**: Затем приложение обращается к API Мегаплана и запрашивает список сделок, созданных после получения этого письма.
4.  **Проверка связи**: Система ищет сделку, контактные данные контрагента в которой (email) совпадают с адресом отправителя письма.
5.  **Определение статуса**:
    *   **Успех**: Если сделка найдена и была создана в течение заданного порогового времени, интеграция считается работающей.
    *   **Предупреждение**: Если сделка найдена, но создана с задержкой, превышающей пороговое значение, система отправляет предупреждение.
    *   **Сбой**: Если в течение порогового времени соответствующая сделка не была найдена, система фиксирует сбой интеграции.

## Логика уведомлений

Система уведомлений разработана так, чтобы предоставлять актуальную информацию и избегать избыточных оповещений:

-   **Уведомление о сбое**: При первом обнаружении проблемы (письмо пришло, а сделка не создалась) система отправляет email-уведомление об ошибке. Чтобы избежать повторных оповещений по одной и той же проблеме, `Message-ID` письма сохраняется, и повторные уведомления по нему блокируются.
-   **Новый сбой**: Если после этого приходит *новое* письмо, которое также не приводит к созданию сделки, система отправит новое уведомление, так как это уже другая проблема.
-   **Уведомление о восстановлении**: Как только система фиксирует первую успешную интеграцию (сделку) после периода сбоев, она отправляет уведомление о восстановлении работы.
-   **"Тихий" режим**: После отправки уведомления о восстановлении система переходит в штатный режим и больше не отправляет уведомлений об успешной работе, пока не произойдет новый сбой.

Состояние системы уведомлений (информация о сбоях и отправленных оповещениях) хранится в файле `notification_state.json`.

## Конфигурация

Все параметры работы приложения настраиваются в файле `appsettings.json`:

-   `MonitoredMailboxes`: Список почтовых ящиков для мониторинга (логин, пароль).
-   `NotificationSettings`: Настройки SMTP-сервера для отправки уведомлений.
-   `MaxTimeToCreateDealAfterLetter`: Максимальное время (в минутах), которое может пройти между получением письма и созданием сделки, чтобы интеграция считалась успешной.
-   `RunIntervalMinutes`: Интервал (в минутах) между циклами проверки.
